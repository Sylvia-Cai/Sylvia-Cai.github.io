一、算法核心思想

BFS（广度优先搜索）

  • 核心思想：层层扩展，先访问离起点最近的节点
  • 使用队列实现
  • 保证按层遍历，先访问距离为1的节点，再距离为2的节点…
  • 适合求最短路径、最少步数等问题

DFS（深度优先搜索）

  • 核心思想：一条路走到底，回溯探索
  • 使用栈实现（递归或显式栈）
  • 优先深入探索分支，直到无法继续再回溯
  • 适合求所有解、连通性、排列组合等问题

方法二：BFS广度优先搜索

  • 解题思路
    让我们用一个更生活化的比喻来理解这个BFS方法，想象你在玩一个数字跳台阶游戏：
    你站在第 n 级台阶上，目标是用最少的步数跳到第 0 级台阶，每次只能往下跳完全平方数级台阶（1级、4级、9级、16级…）
    问：最少需要跳几次？

BFS为什么适合这个问题？
因为BFS会一层一层地探索：
第一层：所有从起点一步能到达的位置
第二层：所有从第一层再走一步能到达的位置
第三层：所有从第二层再走一步能到达的位置
…
当你第一次到达目标（0级台阶）时，所用的步数就是最短路径！

为什么需要visited数组？
避免重复工作和无限循环。例如，如果没有visited数组，数字7会被多次访问：

重复访问不仅浪费计算资源，还可能导致队列无限增长。
优化原理： 每个数字只需要被访问一次，因为第一次访问时找到的就是从起点到该数字的最短路径。

为什么可以提前终止（next < 0时break）？
利用平方数的有序性，即平方数列表是排序的：[1, 4, 9, 16, 25, …]。如果 current - square < 0，那么对于更大的平方数，current - bigger_square 只会更小（更负），所以一旦出现负数，后面的所有平方数都会产生负数，可以直接跳过。

BFS的层数意义
重要理解： level 变量直接对应着”使用了多少个完全平方数”
level = 1：使用1个平方数
level = 2：使用2个平方数
level = 3：使用3个平方数
…

所以当第一次到达0时，level 就是我们需要的最少数量。

一、概述

二分查找是一种高效的搜索算法，时间复杂度为 O(log n)，适用于有序数据集合。本文将系统讲解二分查找的核心思想、实现细节以及各种变种应用。其根本要求是查找的对象处于一个有序的背景下，每一次循环都通过中间值来排除一半，直到找到需要的答案。

二、基础二分查找

其中，有三个需要注意的点：

  • while循环条件为left <= right，代表着循环结束的时候left应该在right的左边（即left = right + 1）。
  • int mid = left + (right - left) / 2 这样来获取中间值不仅可以防止直接进行left+right而导致溢出，实际上还实现了偏左取中间值（向下取整）。我们也可以通过int mid = left + (right - left + 1) / 2来实现偏右取中间值（向上取整）。
  • 二分查找需要确保不进入死循环，基础二分查找需要每次循环至少移动一个边界，保证区间缩小。

七、二分答案法

二分答案法适用于”求满足条件的最小/最大值”类问题，将优化问题转化为判定问题。核心就是根据题目，人为推理得到可能答案的最大值和最小值，然后在这个区间内进行二分查找。二分答案发需要确保答案具有单调性，即如果x可行，那么大于x（或小于x）的值也可行。相比于从可能答案的一端（最小值或者最大值），进行向上或者向下遍历来找到答案，二分答案法的时间复杂度可以从 O(n) 降低到 O(logn)。

力扣刷题笔记：回溯算法及其应用

Posted on 2025-10-14 Edited on 2025-10-31 In 力扣刷题笔记

一、概述

回溯算法是解决组合、排列、搜索类问题的经典方法。本文通过力扣真题，详细讲解回溯算法的核心思想和实现技巧，包含全排列经典问题的解法。

二、回溯算法

回溯算法是一种通过”尝试-回溯”来寻找所有可能解的算法。它通过深度优先搜索的方式遍历解空间树，当发现当前路径不可能得到正确解时，就回溯到上一步尝试其他可能性。

回溯算法通用模板

  • 结束条件： 这是递归的终止点，与具体问题密切相关。比如在全排列中，结束条件是路径长度等于数组长度；在组合总和中，结束条件是当前和等于目标值。

  • for循环： 遍历当前层的所有可能选择。每个循环迭代代表尝试一个候选元素，对应排列中的一个位置。

  • 剪枝判断： 检查当前选择是否合法，如元素是否已使用、是否会导致重复解等，提前排除无效分支，提高算法效率。

  • 做选择与递归： 将当前选择加入路径后，进入下一层递归。每一层递归对应选择排列中的一个位置——第一层选第一个元素，第二层选第二个元素，依此类推。递归就是深度优先搜索的过程。

  • 回溯操作： 当递归到底层满足结束条件（找到一个解）后，程序开始逐层返回。每返回一层，就通过pop()撤销当前层的选择，这样for循环就能继续尝试下一个候选元素。回溯的本质就是 “回头”尝试其他可能性，确保所有路径都被探索到。

简单总结：向下递归是不断做选择，向上回溯是撤销选择换下一个，通过这种机制系统地遍历所有可能解。

三、回溯算法在全排列问题的应用

题目一：全排列（无重复元素）

  1. 题目描述
    给定一个不含重复数字的数组nums，返回其所有可能的全排列。你可以按任意顺序返回答案。

示例：

  1. 题目分析
    这道题是最基础的全排列问题，给定一个不含重复数字的数组 nums，要求返回所有可能的排列。
    核心思路是通过回溯算法模拟“填空过程”：每一层递归选择一个未被使用的数字填入当前位置，直到路径长度等于 nums 的长度为止。

下面结合回溯算法的几个核心组成部分详细分析：

  • 结束条件： 当 path 的长度等于 nums 的长度时，说明所有位置都已填满，形成了一个完整排列。此时应将 path 的拷贝加入结果集。

  • for循环（遍历当前层的选择）： 每一层的循环遍历所有可选元素（即 nums 中的每个数字）。

  • 剪枝判断： 同一个元素在全排序中不能出现两次，所以若当前元素已被使用（used[i] == True），则跳过，防止重复使用相同的数字。

  • 做选择： 将当前数字加入路径 path，并标记为已使用。

  • 递归： 进入下一层递归，继续为排列的下一个位置选择合适的数字。

  • 回溯（撤销选择）： 当递归返回时，说明当前分支的探索已结束，需要撤销本层选择，接下来将继续执行for循环，尝试其他数字。

  1. 完整代码
  1. 整体思路总结：

  • 使用一个布尔数组 used 标记哪些元素已在当前路径中使用。

  • 通过深度优先搜索遍历所有排列路径。

  • 在每次递归返回时撤销选择，实现“回溯”过程。

  • 整个搜索树的叶子节点（路径长度等于 nums 长度）就是一个完整解。

题目二：全排列 II（含重复元素）

  1. 题目描述
    给定一个可包含重复数字的序列 nums ，按任意顺序返回所有不重复的全排列。

示例：

  1. 题目分析
    相比上一题（无重复元素的全排列），这里需要处理重复数字导致的冗余解问题。核心思路仍是回溯，只是在做选择前增加剪枝逻辑以跳过重复情况。
    在开始递归前，先对数组 nums 进行排序，这是去重的关键步骤。排序使得相同数字相邻，便于在循环中通过索引比较发现重复元素。

  • 结束条件： 当路径长度等于 nums 的长度时，说明形成了一个完整排列，加入结果集。（与上一题相同）

  • for循环： 遍历当前层所有未被使用的数字，每次选择一个数字放入路径中。（与上一题相同）

  • 剪枝判断： 如果当前数字 nums[i] 与前一个数字相同，且前一个数字在当前递归层还未被使用 (used[i-1] == False)，说明这两个数字会出现在同一层选择中，产生重复排列，应跳过。（重点理解！）

  • 做选择： 将当前数字加入路径，并标记为已使用 (used[i] = True)。（与上一题相同）

  • 递归： 继续进入下一层，构建下一个位置。（与上一题相同）

  • 回溯： 撤销当前选择（path.pop()），并将 used[i] 恢复为 False，尝试其他可能。（与上一题相同）

其中，比较难理解的是剪枝判断的新条件。通俗地理解，这个条件是在保证重复元素的使用顺序一致：

排序后，相同的数字是紧挨着的。当我们在同一层中选择第二个相同数字（比如第二个 1）时，如果第一个 1 还没被使用，说明我们试图在同一层交换它们的位置，这会生成重复结果。因此，要求“前一个相同数字必须先被使用”，才能使用当前数字。

举例说明：
nums = [1,1,2]
当第一个 1 还没被使用时，不允许使用第二个 1（避免 [1,1,2] 和 [1,1,2] 的重复路径）。只有当第一个 1 已经在更深层使用后，第二个 1 才可以在当前层被选择。

  1. 完整代码
  1. 整体思路总结：
  • 对数组排序，使相同数字相邻，便于去重判断。
  • 通过 used 数组记录哪些数字已在当前路径中使用。
  • 使用剪枝条件避免同层中出现重复选择，确保每种排列唯一。
  • 递归与回溯的过程与无重复版本一致，但多了对重复元素的控制。

四、回溯算法在组合问题的应用

题目一：组合总和（无重复元素，可重复使用）

  1. 题目描述
    给你一个无重复元素的整数数组 candidates 和一个目标整数 target，找出 candidates 中可以使数字和为目标数 target 的所有不同组合。candidates 中的同一个数字可以无限制重复被选取。

示例：

  1. 题目分析
    这道题是组合问题的经典变种，与全排列问题的主要区别在于：
  • 元素顺序不重要：[2,2,3] 和 [2,3,2] 被视为相同组合
  • 元素可重复使用：同一个数字可以被多次选取
  • 需要控制起始位置：避免生成重复组合
  1. 核心思路：
  • 使用 start 参数控制选择范围的起始位置，确保不会回头选择之前的元素。具体来说，只会生成[2,2,3]，确保不会生成 [2,3,2] 这样的重复组合，因为每次从 start 开始，不会回头选择之前的元素。
  • 不需要 used 数组，因为通过 start 参数已经控制了选择范围。
  • 递归时传递 i 而不是 i+1（backtrack(i, path, current_sum)），允许重复使用当前元素。
  • 当当前和等于目标值时记录结果，超过目标值时提前返回。

题目二：组合总和 II（含重复元素，不可重复使用）

  1. 题目描述
    给定一个候选人编号的集合 candidates 和一个目标数 target，找出 candidates 中所有可以使数字和为 target 的组合。candidates 中的每个数字在每个组合中只能使用一次，且解集不能包含重复的组合。

示例：

  1. 题目分析
    这道题相比上一题增加了两个约束：
  • 元素只能使用一次
  • 数组中可能包含重复元素
  1. 解决方案：
    排序：先对数组排序，使相同元素相邻
    跳过重复元素：在同一层循环中，如果当前元素与前一个相同且前一个未被使用，则跳过

方法一：使用used数组（直观理解）

这种方法沿用了全排列II的思路，使用used数组来标记元素是否被使用。去重条件if i > 0 and candidates[i] == candidates[i-1] and not used[i-1]的含义是：如果当前元素与前一个元素相同，且前一个元素在当前递归层还没有被使用，那么就跳过当前元素。

这种方法的优点是逻辑清晰，与排列问题的解法保持一致，但需要维护额外的used数组。

方法二：简化版本

这是一种更优雅的解法，利用了组合问题的特性。关键的去重条件if i > start and candidates[i] == candidates[i-1]: continue
其中：

  • i > start：确保我们只在同一层中跳过重复元素
  • candidates[i] == candidates[i-1]：当前元素与前一个元素相同
  • continue：跳过当前元素

为什么这个方法有效？
当i == start时，即使当前元素与前一个相同，我们也不跳过，因为这是新分支的开始
当i > start时，如果当前元素与前一个相同，说明在同一层中已经考虑过这个值了，应该跳过

举例说明：
对于排序后的数组[1,1,2,5,6,7,10]和target=8：

  • 第一层循环i=0，选择第一个1，进入递归
  • 第二层循环i=1，选择第二个1（因为i=1 > start=1不成立，所以不跳过）
  • 当回到第一层循环i=1时，i=1 > start=0成立，且第二个1与第一个1相同，所以跳过

这种方法省去了used数组，代码更简洁，是解决组合去重问题的标准写法。

五、组合问题与排列问题的对比

特性 组合问题 排列问题
元素顺序 不重要 重要
重复判断 使用start参数 使用used数组
去重方式 跳过同一层重复元素 跳过未使用的重复元素
递归参数 传递i或i+1 总是从0开始遍历

六、总结

回溯算法作为解决组合优化问题的利器，其核心在于”尝试-回溯”的思维模式。通过系统性地探索解空间，并在适当时候剪枝，回溯算法能够高效地找到所有可行解。在掌握了基本模板后，真正的精髓在于根据不同问题的特性进行灵活调整。

核心技巧精要：

  • 排序定基础：处理含重复元素的问题时，排序是去重的前提。有序数组让相同元素相邻，为后续剪枝创造条件，这是解决去重问题的第一步也是关键一步。

  • 状态管理分情况：排列问题用used数组记录使用状态，组合问题用start参数控制选择范围。理解这个区别是避免重复解的关键——排列关心顺序，需要知道每个元素是否用过；组合不关心顺序，只需保证不回头选择。

  • 去重逻辑抓本质：排列去重关注”前一个相同元素是否未被使用”（not used[i-1]），防止同一层出现相同选择；组合去重关注”是否在同一层且与前一个相同”（i > start），保证每个值在同一层只选一次。这种细微差别正是问题本质的体现。

  • 参数传递见意图：递归时传递i允许重复使用当前元素，传递i+1禁止重复使用，传递start保持选择范围。这个简单的参数变化直接体现了问题的约束条件。

  • 剪枝时机抢效率：在组合总和中，当current_sum > target时立即返回，这种提前终止能显著减少不必要的递归调用。好的剪枝策略能让算法效率提升数个量级。

  • 路径拷贝保结果：使用path[:]记录结果而非直接引用，这是避免后续操作污染已发现解的保障措施，虽是小细节，却关乎结果正确性。
