力扣刷题笔记：BFS与DFS算法及应用

Posted on 2025-11-10 In 力扣刷题笔记

一、算法核心思想

BFS（广度优先搜索）

  • 核心思想：层层扩展，先访问离起点最近的节点
  • 使用队列实现
  • 保证按层遍历，先访问距离为1的节点，再距离为2的节点…
  • 适合求最短路径、最少步数等问题

BFS代码模版（采用队列）

DFS（深度优先搜索）

  • 核心思想：一条路走到底，回溯探索
  • 使用栈实现（递归或显式栈）
  • 优先深入探索分支，直到无法继续再回溯
  • 适合求所有解、连通性、排列组合等问题

二、典型题目

题目一：1091. 二进制矩阵中的最短路径

  • 题目分析
    这道题要求在一个二进制矩阵中找出从左上角到右下角的最短畅通路径。其中畅通路径需要满足两个条件：路径上的所有单元格值都是0，且相邻单元格可以在8个方向上连通。由于是求最短路径问题，这天然适合使用BFS算法来解决，因为BFS会按层扩展，首先找到的路径一定是最短的。

  • 解题思路

  1. 起点或终点为1时直接返回-1

  2. 使用BFS按层扩展，记录每个位置的最短距离

  3. 8个方向探索，遇到边界或障碍物跳过

  4. 到达终点立即返回当前步数

方法二：BFS广度优先搜索

  • 解题思路
    让我们用一个更生活化的比喻来理解这个BFS方法，想象你在玩一个数字跳台阶游戏：
    你站在第 n 级台阶上，目标是用最少的步数跳到第 0 级台阶，每次只能往下跳完全平方数级台阶（1级、4级、9级、16级…）
    问：最少需要跳几次？

BFS为什么适合这个问题？
因为BFS会一层一层地探索：
第一层：所有从起点一步能到达的位置
第二层：所有从第一层再走一步能到达的位置
第三层：所有从第二层再走一步能到达的位置
…
当你第一次到达目标（0级台阶）时，所用的步数就是最短路径！

为什么需要visited数组？
避免重复工作和无限循环。例如，如果没有visited数组，数字7会被多次访问：

重复访问不仅浪费计算资源，还可能导致队列无限增长。
优化原理： 每个数字只需要被访问一次，因为第一次访问时找到的就是从起点到该数字的最短路径。

为什么可以提前终止（next < 0时break）？
利用平方数的有序性，即平方数列表是排序的：[1, 4, 9, 16, 25, …]。如果 current - square < 0，那么对于更大的平方数，current - bigger_square 只会更小（更负），所以一旦出现负数，后面的所有平方数都会产生负数，可以直接跳过。

BFS的层数意义
重要理解： level 变量直接对应着”使用了多少个完全平方数”
level = 1：使用1个平方数
level = 2：使用2个平方数
level = 3：使用3个平方数
…

所以当第一次到达0时，level 就是我们需要的最少数量。

题目三：200. 岛屿数量

  • 题目分析
    这道题要求计算二维网格中岛屿的数量，其中岛屿由相邻的陆地（’1’）组成，相邻指的是水平或垂直方向相邻。这是一个典型的连通分量计数问题，适合使用DFS或BFS来标记和统计连通区域。每次遇到一个未访问的陆地单元格，就启动一次DFS/BFS来标记整个相连的岛屿区域。本题在标记陆地是否访问过的时候有两种办法，第一种是新建一个visited的二维数组，标记每一块陆地的访问状态。第二种是直接在原数据上修改，访问过的陆地可以直接设置为0，这样也就相当于这块陆地变成了水域，不会影响后续陆地的计算。类似题目还有695. 岛屿的最大面积；463. 岛屿的周长。

  • 解题思路

  1. 遍历网格中的每个单元格，当遇到值为’1’且未被访问的单元格时，说明发现了一个新的岛屿，此时岛屿计数加1。
  2. 然后使用DFS递归标记所有与该单元格相连的陆地单元格为已访问。
  3. DFS过程中，对于每个陆地单元格，检查其上下左右四个方向的相邻单元格，如果是陆地且未访问，则递归访问。这样确保整个岛屿的所有单元格都被标记，避免重复计数。

题目四：463. 岛屿的周长

  • 题目分析
    这道题要求计算一个二维网格中单个岛屿的周长。岛屿由相邻的陆地单元格（值为1）组成，相邻指的是水平或垂直方向相邻（不包括对角线）。题目保证网格中恰好只有一个岛屿，且岛屿内部没有”湖”（即水域完全包围岛屿，岛屿内部没有独立的水域）。每个单元格是边长为1的正方形，因此每条边都贡献1单位周长。我们可以用DFS来计算。

  • 解题思路

  1. DFS遍历岛屿：使用深度优先搜索来遍历整个岛屿的所有陆地单元格
  2. 边界检测：对于每个陆地单元格，检查其四个方向的相邻位置：
    如果相邻位置是网格边界或水域，则该边贡献1单位周长
    如果相邻位置是未访问的陆地，则继续DFS遍历
  3. 标记访问：为避免重复计算，将访问过的陆地标记为特殊值（如-1）

题目五：547. 省份数量

  • 题目分析
    这道题考察的是无向图中连通分量的数量。城市之间的连接关系用邻接矩阵表示，isConnected[i][j] = 1表示城市i和城市j直接相连。省份定义为直接或间接相连的一组城市。需要找出矩阵中省份的数量，即连通分量的个数。

  • 解题思路

  1. 图遍历：将城市看作图中的节点，连接关系看作边
  2. DFS搜索：从每个未访问的城市开始DFS，标记所有连通的城市
  3. 计数省份：每次新的DFS开始，就意味着发现了一个新的省份
  4. 访问标记：使用visited数组避免重复访问
