力扣刷题笔记：二分查找算法及其变形

Posted on 2025-10-29 Edited on 2025-10-30 In 力扣刷题笔记

一、概述

二分查找是一种高效的搜索算法，时间复杂度为 O(log n)，适用于有序数据集合。本文将系统讲解二分查找的核心思想、实现细节以及各种变种应用。其根本要求是查找的对象处于一个有序的背景下，每一次循环都通过中间值来排除一半，直到找到需要的答案。

二、基础二分查找

704. 二分查找

问题描述：在有序数组中查找目标值，存在则返回下标，否则返回 -1。

其中，有三个需要注意的点：

  • while循环条件为left <= right，代表着循环结束的时候left应该在right的左边（即left = right + 1）。
  • int mid = left + (right - left) / 2 这样来获取中间值不仅可以防止直接进行left+right而导致溢出，实际上还实现了偏左取中间值（向下取整）。我们也可以通过int mid = left + (right - left + 1) / 2来实现偏右取中间值（向上取整）。
  • 二分查找需要确保不进入死循环，基础二分查找需要每次循环至少移动一个边界，保证区间缩小。

第二种是左边界可能保持不变，配套使用偏右取中间值（int mid = left + (right - left + 1) / 2）可以避免进入死循环。

这种题目的关键是，看当前mid所对应的值需不需要保留在区间内，如果需要的话，就需要使用 right = mid 或者 left = mid。另外，注意边界查找问题所使用的二分查找的循环条件为left < right，循环结束时left = right = mid。这样做是为了避免当 right = mid 且 mid == right 时的死循环

33. 搜索旋转排序数组

问题分析
目标：在旋转数组中查找目标值target，返回其索引，不存在则返回-1
约束：必须使用O(log n)时间复杂度算法

关键观察：
虽然整个数组不是完全有序，但至少有一半是有序的。我们可以判断哪一半是有序的，然后确定目标值在哪一半。

153. 寻找旋转排序数组中的最小值

关键判断：nums[mid] 与 nums[right] 的比较
情况1：nums[mid] <= nums[right]
说明右半部分 [mid, right] 是有序的，最小值不可能在 [mid, right] 范围内（因为这部分是有序递增的），最小值可能在 [left, mid] 范围内。

情况2：nums[mid] > nums[right]
说明左半部分 [left, mid] 是有序的，最小值不可能在 [left, mid] 范围内（因为这部分是有序递增的），最小值一定在 [mid, right] 范围内。

五、峰值查找问题

  • 问题特点：数组可能包含多个峰值，找到任意一个即可
  • 核心思想：沿着上升的方向搜索，因为峰值一定在上升序列的尽头
  • 数学原理：
    如果 nums[mid] < nums[mid+1]，说明右侧存在更大的值，根据边界条件 nums[n] = -∞，右侧一定有峰值
    如果 nums[mid] > nums[mid+1]，说明峰值在左侧或当前位置，根据边界条件 nums[-1] = -∞，左侧一定有峰值

六、二维矩阵搜索

在二维矩阵中查找特定的值，这个二维矩阵有一定的特征，一般来说是全部有序或者部分有序。

74. 搜索二维矩阵

核心思想：将二维矩阵视为一维有序数组进行二分查找
坐标转换：行索引：mid / columns；列索引：mid % columns。
前提条件：矩阵必须满足每行的第一个整数大于前一行的最后一个整数

240. 搜索二维矩阵 II（Z字形搜索）

问题特点：每行从左到右升序，每列从上到下升序
核心思想：从右上角开始，利用行列的有序性逐步缩小搜索范围
搜索策略：
如果当前值 < target：排除当前行（向下移动）
如果当前值 > target：排除当前列（向左移动）

七、二分答案法

二分答案法适用于”求满足条件的最小/最大值”类问题，将优化问题转化为判定问题。核心就是根据题目，人为推理得到可能答案的最大值和最小值，然后在这个区间内进行二分查找。二分答案发需要确保答案具有单调性，即如果x可行，那么大于x（或小于x）的值也可行。相比于从可能答案的一端（最小值或者最大值），进行向上或者向下遍历来找到答案，二分答案法的时间复杂度可以从 O(n) 降低到 O(logn)。

875. 爱吃香蕉的珂珂

  • 题目描述：
    珂珂喜欢吃香蕉。这里有 n 堆香蕉，第 i 堆中有 piles[i] 根香蕉。警卫将在 h 小时后回来。珂珂可以决定吃香蕉的速度 k（根/小时）。每个小时，她选择一堆香蕉，吃掉 k 根。如果这堆少于 k根，她将吃掉这堆的所有香蕉，然后这一小时内不会再吃更多的香蕉。返回她可以在 h 小时内吃掉所有香蕉的最小速度 k。
  • 关键点：
    吃的速度 k 越大，所需时间越少
    吃的速度 k 越小，所需时间越多
    我们需要找到最小的 k，使得总时间 ≤ h
  • 答案范围：
    最小速度：1根/小时（最慢）
    最大速度：max(piles)根/小时（最快，一次吃完最大堆）

1011. 在 D 天内送达包裹的能力

  • 题目描述
    传送带上的包裹必须在 days 天内从一个港口运送到另一个港口。传送带上的第 i个包裹的重量为 weights[i]。每一天，按给出重量的顺序往传送带上装载包裹。装载的重量不会超过船的最大运载重量。返回能在 days 天内将传送带上的所有包裹送达的船的最低运载能力。

  • 关键点：
    包裹必须按顺序装载
    每天的装载重量不能超过船的运载能力
    需要找到最小的运载能力，使得能在days天内运完

  • 答案范围：
    最小能力：max(weights)（至少能装下最重的包裹）
    最大能力：sum(weights)（一次装完所有包裹）
